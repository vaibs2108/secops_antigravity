# AGENTS.md
GenAI Security Operations Demonstration Platform

------------------------------------------------------------
1. Project Objective
------------------------------------------------------------

Build a GenAI-powered Security Operations demonstration platform that simulates enterprise-scale cybersecurity environments using synthetic data generated dynamically at runtime.

The system demonstrates how specialized AI agents improve cybersecurity operations across multiple domains while providing complete transparency into how the AI works.

The platform must NOT behave like a black box. Users must be able to see:

• Input datasets used in demos (Contextual Preview)
• Processing steps performed by AI agents (Status Logs)
• Outputs generated by the system (Action Plans)
• AI reasoning and technical findings (Analysis)
• KPI impact after AI intervention (Before vs After)

UI AESTHETIC CONSTRAINTS:
The platform must utilize a premium, "world-class" Executive Enterprise SecOps design. This includes:
- Modern, clean Light Theme with high-contrast corporate aesthetics.
- Background: #F8FAFC (Slate-White) with subtle Royal Blue / Navy gradients.
- Custom Google typography: 'Outfit' (Sans-serif).
- Executive Dashboard: Premium high-contrast headers with 135deg royal blue gradients, white text, and glassmorphism icon boxes.
- Metric Cards: Multi-layered shadows, Navy (#1E3A8A) values, and white backgrounds.
- High-contrast anomaly grids (Dataframes) with specific, granular row data (IPs, Hostnames, User IDs).
- Floating components and smooth transitions for an interactive command-center feel.

Deployment targets: HuggingFace Spaces (Git/Docker based).

3. Synthetic Data Generation & RAG
------------------------------------------------------------

The platform generates two distinct types of synthetic data at runtime:

I. Global Performance Telemetry (generator.py)
   - Assets (CMDB): 10,000+ (with realistic 10.x.x.x IPs and hostnames)
   - Alerts: 20,000+
   - Incidents: 500+
   - ITIL Data: Identity (IAM), Config Baselines, HR Workday data
   - Observability: 20,000 NetFlow/Trace events (Gigamon/Packet telemetry)
   - Specialized Stores: EDR Telemetry, Playbooks, Threat Models (MITRE), Git Logs, RCA Documents
   - Financial Data: 200 Corporate Card & PO Transactions (for Shadow IT demos)

II. Operational Knowledge Base (rag_generator.py)
   - KEDB Entries: 50 technical records (KE ID, Symptoms, Root Cause, Fix).
   - ITIL Tickets: 60 sample records (CHG, INC, SR IDs).

HYBRID SEARCH ARCHITECTURE (SECopsRAGEngine):
1. Regex-based extraction for alphanumeric IDs (KE-, CHG, INC, SR).
2. Exact-match retrieval from metadata for high-precision ticket lookup.
3. Semantic Vector Similarity (FAISS) for broad reasoning queries.
4. Omni-Dataset Injection: Injecting 5-row snapshots of 8+ tables into the AI context for holistic awareness.

10. Guardrails & Safety Architecture
------------------------------------------------------------

Deterministic Safety Constraints:
- Human-in-the-loop: Required for all disruptive containment (Major Incidents).

- Data Anonymization: Masking PII/Sensitive data before LLM transmission.
- Logical Checkpoints: Agents must validate compliance baseline prior to execution.
- LLM Output Validation: Pydantic-enforced JSON schema to eliminate hallucination.

13. AI Execution & Analysis Engine (Live Generative Logic)
------------------------------------------------------------

The platform implements a "Zero-Mock" AI Execution model. Demos are NOT pre-scripted; they are live generative analyses of the synthetic environment.

I. Dynamic Context Slicing
   - When a demo is triggered, the system identifies "Relevant Tables" (e.g., EDR + Asset + Identity).
   - It performs a targeted slice (Top 50 rows) from the 10,000+ row synthetic databases to ensure LLM context window efficiency.

II. LangChain Prompt Injection
   - The CSV-formatted slice is injected directly into the `SystemMessage` prompt.
   - The prompt explicitly commands the AI: "Parse the attached dataset. Identify specific anomalies (IPs/Hostnames) present in this exact slice."

III. Structured Output Generation (Pydantic)
   - The LLM outcome is enforced via a Pydantic `SecOpsOutcome` model.
   - The engine extracts:
     1. Textual Analysis (Reasoning)
     2. Recommended Action Plan (Bullet points)
     3. Anomaly Data Grid (A JSON array of specific matching rows from the synthetic data)
     4. Quantified Metrics (Confidence % and Anomaly Counts)

IV. Real-time Analysis Rendering
   - Section 4 of every demo renders this live-generated JSON payload. This ensures that every time a user "Regenerates Dataset," the AI analysis uniquely changes to reflect the new random distribution of data.

14. Executive Authentication & User Persona
------------------------------------------------------------

The platform includes a robust, isolated Executive Command Center Authentication portal prior to granting access to the synthetic datasets or the AI agents.

I. Secure Configuration
   - The authentication credentials (`ADMIN_USERNAME` and `ADMIN_PASSWORD`) are not hardcoded.
   - They are ingested securely via the local environment `.env` file (or Hugging Face Space secrets) at runtime, allowing dynamic updates without touching the codebase.

II. Glassmorphic Landing Page
   - Unauthenticated users land on a meticulously styled `[data-testid="stForm"]` card centered over a custom global AI security background.
   - The application intercepts the streamlit rendering loop (`st.stop()`) to completely hide the navigation sidebar and upper headers until successful `.env` credential validation.
   - Loading Transition: A custom HTML interstitial loader animates while the heavy synthetic data/vector databases initialize in the background, preventing 'stale UI glitches'.

III. Dynamic Persona Widget
   - Post-authentication, a global, floating HTML/CSS widget is injected into the top-right corner of the application (`z-index: 999999`).
   - This widget dynamically pulls the active username (e.g., "Admin") and their title ("SecOps Administrator"), featuring a custom initials badge styled with a premium background gradient.
   - A convenient "Logout" button is appended to the bottom of the navigation sidebar allowing seamless state clearing and session invalidation.

15. Agent Transparency & Traceability Ledger
------------------------------------------------------------

To eliminate "Black Box AI" experiences, the platform enforces native Under-the-Hood execution tracing for all autonomous agents:

I. Agent Directory Capabilities mapping
   - `agent_console.py` natively parses the `Goal of Demo` from the `demo_inputs.json` to generate explicit "Key Capabilities".
   - The global DataFrame tracks distinct "Under-the-Hood Tasks" defining precisely how a specific agent acts (e.g., "Ransomware Kill Switch: Broadcasts API halt commands").

II. Persistent Multi-Agent Trace
   - Above Section 1 in any demo, an autonomous Multi-Agent Collaboration Plan is permanently published highlighting exact step-by-step responsibilities across different personas.

III. Real-Time Execution Ledger
   - Global dashboard state arrays record identically which `Active AI Agents` were dispatched, resolving ambiguity and capturing audit records of every simulation executed.
